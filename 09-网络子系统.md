# 网络子系统分析

## 一、systemd-networkd — 网络配置管理

### 1.1 概述

`systemd-networkd` 是网络配置守护进程，管理网络链路、地址、路由等。支持 DHCP、静态配置、虚拟网络设备等。

D-Bus 名称：`org.freedesktop.network1`
源码位于：`src/network/`

### 1.2 核心架构

```
┌──────────────────────────────────────────────────┐
│               systemd-networkd                    │
│                                                   │
│  Manager                                          │
│  ├── rtnetlink (路由 netlink)                     │
│  │     监听链路/地址/路由变更                        │
│  ├── genetlink (通用 netlink)                     │
│  │     监听 WiFi 等事件                            │
│  ├── udev monitor                                 │
│  │     监听设备事件                                 │
│  ├── D-Bus                                        │
│  │     对外暴露管理接口                              │
│  └── 对象管理                                      │
│       ├── Link (链路)                             │
│       ├── Network (.network 配置)                 │
│       ├── NetDev (.netdev 虚拟设备)               │
│       ├── Address (地址)                          │
│       ├── Route (路由)                            │
│       └── Rule (策略路由规则)                      │
└──────────────────────────────────────────────────┘
```

### 1.3 关键数据结构

#### Manager（networkd-manager.h:20-104）

```c
struct Manager {
    sd_netlink *rtnl;              // 路由 netlink
    sd_netlink *genl;              // 通用 netlink
    sd_event *event;               // 事件循环
    sd_bus *bus;                   // D-Bus 连接
    sd_device_monitor *device_monitor; // 设备监控

    Hashmap *links;                // 链路集合（ifindex → Link）
    Hashmap *links_by_name;        // 链路集合（name → Link）
    Hashmap *netdevs;              // 虚拟设备
    OrderedHashmap *networks;      // 网络配置

    // DNS/NTP 配置
    OrderedSet *dns;               // DNS 服务器
    OrderedSet *ntp;               // NTP 服务器
    OrderedSet *search_domains;    // 搜索域
    OrderedSet *route_domains;     // 路由域

    // 状态
    AddressPool *address_pools;    // 地址池
    Set *rules;                    // 路由规则
    ...
};
```

#### Link（networkd-link.h:47-194）

```c
typedef struct Link {
    Manager *manager;
    int ifindex;                   // 接口索引
    char *ifname;                  // 接口名
    unsigned short iftype;         // 接口类型
    LinkOperationalState operstate; // 操作状态

    Network *network;              // 关联的 .network 配置

    // DHCP 引擎
    sd_dhcp_client *dhcp_client;
    sd_dhcp_lease *dhcp_lease;
    sd_dhcp6_client *dhcp6_client;
    sd_ipv4ll *ipv4ll;             // IPv4LL 自动配置
    sd_ndisc *ndisc;               // IPv6 邻居发现
    sd_radv *radv;                 // IPv6 路由器通告
    sd_lldp *lldp;                 // LLDP 协议

    // 地址/路由/邻居
    Set *addresses;
    Set *routes;
    Set *neighbors;

    // DNS 配置
    struct in_addr_full **dns;
    char **ntp;
    char **search_domains;
    ...
} Link;
```

### 1.4 配置文件

#### .network 文件（匹配和配置网络接口）

```ini
# /etc/systemd/network/10-eth0.network
[Match]
Name=eth0

[Network]
DHCP=yes
DNS=8.8.8.8
NTP=pool.ntp.org
Domains=example.com

[Address]
Address=192.168.1.100/24

[Route]
Gateway=192.168.1.1
Destination=10.0.0.0/8
```

#### .netdev 文件（虚拟网络设备）

```ini
# /etc/systemd/network/25-bridge.netdev
[NetDev]
Name=br0
Kind=bridge

# 支持的虚拟设备类型:
# bridge, bond, vlan, macvlan, ipvlan, vxlan,
# veth, tun, tap, dummy, vrf, wireguard, geneve, l2tp 等
```

#### .link 文件（底层链路配置）

```ini
# /etc/systemd/network/10-dmz.link
[Match]
MACAddress=00:a0:de:63:7a:e6

[Link]
Name=dmz0
MTUBytes=1500
```

### 1.5 链路配置流程

```
udev 检测到网络设备
  ↓
networkd 通过 rtnetlink 收到 NEWLINK 事件
  ↓
manager_process_link()
  ├── 创建 Link 对象
  ├── 匹配 .network 配置文件
  │     按 [Match] 段匹配（Name、MAC、Type 等）
  ├── link_configure()
  │     ├── 设置 MTU、MAC 地址等底层属性
  │     ├── 启动 DHCP 客户端（如果配置）
  │     ├── 配置静态地址/路由
  │     ├── 启动 IPv6 RA/NDISC
  │     └── 配置 DNS/NTP/Domains
  └── link_set_state(LINK_STATE_CONFIGURED)
```

## 二、systemd-resolved — DNS 解析

### 2.1 概述

`systemd-resolved` 提供 DNS 解析服务，包括：
- 本地 DNS 存根监听（127.0.0.53:53）
- DNS 缓存
- DNSSEC 验证
- LLMNR/mDNS 支持
- 多链路 DNS 路由

D-Bus 名称：`org.freedesktop.resolve1`
源码位于：`src/resolve/`

### 2.2 架构

```
┌───────────────────────────────────────────────┐
│             systemd-resolved                   │
│                                                │
│  ┌─────────────────────────────────┐          │
│  │ Stub Listener (127.0.0.53:53)  │          │
│  │ → 接收本地 DNS 查询             │          │
│  └──────────────┬──────────────────┘          │
│                 ▼                              │
│  ┌─────────────────────────────────┐          │
│  │ DNS Cache (缓存层)              │          │
│  │ → 命中则直接返回                 │          │
│  └──────────────┬──────────────────┘          │
│                 ▼                              │
│  ┌─────────────────────────────────┐          │
│  │ Scope Router (域路由)           │          │
│  │ → 按域名选择链路/DNS 服务器      │          │
│  └──────────────┬──────────────────┘          │
│                 ▼                              │
│  ┌────────┬────────┬────────┬────────┐        │
│  │  DNS   │ LLMNR  │  mDNS  │ DNS/TLS│        │
│  │上游查询 │本地链路 │多播DNS │加密DNS  │        │
│  └────────┴────────┴────────┴────────┘        │
└───────────────────────────────────────────────┘
```

### 2.3 Manager 结构体（resolved-manager.h:35-156）

```c
struct Manager {
    sd_event *event;
    sd_bus *bus;

    // DNS 配置
    DnsServer *dns_servers;              // 系统级 DNS 服务器
    DnsServer *fallback_dns_servers;     // 回退 DNS 服务器
    LIST_HEAD(DnsSearchDomain, search_domains); // 搜索域

    // 链路管理
    Hashmap *links;                      // 网络链路

    // 缓存与事务
    DnsCache cache;                      // DNS 缓存
    LIST_HEAD(DnsTransaction, transactions); // 活跃事务
    LIST_HEAD(DnsStream, streams);       // TCP 流

    // 本地监听
    int dns_stub_listener_fd;            // 存根监听 fd

    // DNSSEC
    ResolveSupport dnssec_mode;          // DNSSEC 模式

    // /etc/hosts
    EtcHosts *etc_hosts;                 // hosts 文件缓存

    // Varlink
    VarlinkServer *varlink_server;       // Varlink 服务
    ...
};
```

### 2.4 配置文件

```ini
# /etc/systemd/resolved.conf
[Resolve]
DNS=8.8.8.8 8.8.4.4
FallbackDNS=1.1.1.1
Domains=~.                      # 路由所有域名
DNSSEC=allow-downgrade
DNSOverTLS=opportunistic
LLMNR=yes
MulticastDNS=yes
Cache=yes
CacheFromLocalhost=no
DNSStubListener=yes
```

### 2.5 /etc/resolv.conf 管理

resolved 管理 `/etc/resolv.conf`，提供多种模式：

| 模式 | 文件内容 |
|------|---------|
| stub | nameserver 127.0.0.53（推荐） |
| static | nameserver 配置的上游 DNS |
| uplink | 直接使用上游 DNS |

## 三、systemd-timesyncd — NTP 时间同步

### 3.1 概述

轻量级 NTP 客户端（仅客户端，非服务端），使用 SNTP 协议。

D-Bus 名称：`org.freedesktop.timesync1`
源码位于：`src/timesync/`

### 3.2 Manager 结构体（timesyncd-manager.h:37-112）

```c
struct Manager {
    sd_event *event;
    sd_bus *bus;
    sd_resolve *resolve;           // 异步 DNS 解析

    // 服务器管理
    LIST_HEAD(ServerName, system_servers);    // 系统配置服务器
    LIST_HEAD(ServerName, link_servers);      // 链路配置服务器
    LIST_HEAD(ServerName, fallback_servers);  // 回退服务器
    ServerName *current_server_name;          // 当前使用的服务器

    // 同步状态
    struct timespec trans_time;     // 上次发送时间
    usec_t poll_interval_usec;     // 轮询间隔
    bool poll_resync;              // 需要重新同步

    // 样本/偏差
    struct {
        double offset;
        double delay;
        double dispersion;
        double root_distance;
    } samples[8];                  // 采样历史
    ...
};
```

### 3.3 同步流程

```
启动
  ├── 从 /var/lib/systemd/timesync/clock 恢复时钟
  ├── 选择 NTP 服务器（系统 > 链路 > 回退）
  ├── 解析服务器地址
  └── 开始同步循环
       ├── 发送 NTP 请求
       ├── 接收响应
       ├── 计算偏差
       ├── adjtime() 调整系统时钟
       ├── 更新 /var/lib/systemd/timesync/clock
       └── 等待下一轮（poll_interval 自适应）
```

### 3.4 配置文件

```ini
# /etc/systemd/timesyncd.conf
[Time]
NTP=ntp.example.com
FallbackNTP=0.pool.ntp.org 1.pool.ntp.org
RootDistanceMaxSec=5
PollIntervalMinSec=32
PollIntervalMaxSec=2048
```
