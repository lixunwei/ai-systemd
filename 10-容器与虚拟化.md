# 容器与虚拟化分析

## 一、systemd-nspawn — 容器运行工具

### 1.1 概述

`systemd-nspawn` 是一个轻量级容器运行工具，类似于 `chroot` 的增强版，提供完整的 Linux 命名空间隔离。常用于系统开发、测试和构建。

源码位于：`src/nspawn/`

### 1.2 核心文件

| 文件 | 职责 |
|------|------|
| nspawn.c | 主入口，参数解析，容器启动逻辑 |
| nspawn-settings.h | Settings 结构体和配置选项定义 |
| nspawn-settings.c | 配置文件加载 |
| nspawn-mount.c | 容器内挂载管理 |
| nspawn-network.c | 容器网络配置 |
| nspawn-expose-ports.c | 端口映射 |
| nspawn-cgroup.c | cgroup 设置 |
| nspawn-register.c | machined 注册 |
| nspawn-seccomp.c | seccomp 过滤器 |
| nspawn-oci.c | OCI 容器规范支持 |
| nspawn-patch-uid.c | UID 映射 |

### 1.3 Settings 结构体（nspawn-settings.h:162-237）

```c
typedef struct Settings {
    // 启动模式
    StartMode start_mode;              // boot/as-pid2/exec
    char **parameters;                 // 启动参数

    // 用户命名空间
    UserNamespaceMode userns_mode;     // off/pick/fixed
    uid_t uid_shift;
    uid_t uid_range;

    // 网络
    NetworkNamespaceMode private_network; // 私有网络
    char **network_interfaces;          // 移入容器的接口
    char **network_macvlan;            // MACVLAN 接口
    char **network_ipvlan;             // IPVLAN 接口
    char **network_veth_extra;         // 额外 veth 对
    int network_veth;                  // 是否创建 veth

    // 文件系统
    char *root;                        // 根文件系统
    MountEntry *custom_mounts;         // 自定义挂载
    VolatileMode volatile_mode;        // 易失模式

    // 安全
    Capability capability;             // 授予的 capabilities
    SeccompMode seccomp;               // seccomp 模式

    // 资源解析
    ResolvConfMode resolv_conf;        // resolv.conf 处理
    TimezoneMode timezone;             // 时区处理
    LinkJournal link_journal;          // 日志链接模式
    ...
} Settings;
```

### 1.4 容器启动流程

```
systemd-nspawn --boot -D /path/to/rootfs
  ↓
解析参数 → 加载 Settings
  ↓
clone(CLONE_NEWNS | CLONE_NEWPID | CLONE_NEWNET | ...)
  ↓
父进程（外部）:
  ├── 注册到 machined（D-Bus）
  ├── 设置 cgroup
  ├── 设置网络（veth 等）
  ├── 转发端口
  └── 等待子进程
  
子进程（容器内）:
  ├── mount_all() — 挂载 /proc, /sys, /dev 等
  ├── 设置 /etc/resolv.conf
  ├── 设置时区
  ├── 链接日志（可选）
  ├── 应用 seccomp 过滤器
  ├── 降低 capabilities
  └── exec() — 启动 PID 1（--boot）或指定命令
```

### 1.5 命名空间隔离

| 命名空间 | 用途 |
|---------|------|
| mount | 独立的文件系统挂载 |
| PID | 独立的进程 ID 空间（容器内 PID 1） |
| network | 独立的网络栈（可选） |
| UTS | 独立的主机名 |
| IPC | 独立的 IPC 空间 |
| user | 用户 ID 映射（可选） |
| cgroup | 独立的 cgroup 视图 |

## 二、systemd-machined — 虚拟机/容器注册中心

### 2.1 概述

`systemd-machined` 是虚拟机和容器的注册管理服务，跟踪所有运行中的机器实例。

D-Bus 名称：`org.freedesktop.machine1`
源码位于：`src/machine/`

### 2.2 核心数据结构

#### Manager（machined.h:19-44）

```c
struct Manager {
    sd_event *event;
    sd_bus *bus;

    Hashmap *machines;           // 所有注册的机器
    Hashmap *machine_units;      // 机器 → 单元映射
    Hashmap *machine_leaders;    // 机器 → leader PID

    Machine *host_machine;       // 宿主机器

    LIST_HEAD(Operation, operations); // 异步操作队列

    Hashmap *image_cache;        // 镜像缓存
    sd_event_source *image_cache_defer_event; // 缓存失效
    ...
};
```

#### Machine（machine.h:35-67）

```c
struct Machine {
    Manager *manager;
    char *name;                  // 机器名
    sd_id128_t id;              // 机器 ID

    MachineClass class;          // 类型：container/vm/host
    MachineState state;          // 状态：opening/running/closing

    char *scope_job;             // 关联的 scope Job
    char *unit;                  // 关联的 systemd 单元
    char *root_directory;        // 根目录

    pid_t leader;                // leader 进程 PID
    int *netif;                  // 网络接口列表
    size_t n_netif;              // 网络接口数量

    dual_timestamp timestamp;    // 创建时间
    ...
};
```

### 2.3 机器类型

| 类型 | 含义 | 注册方 |
|------|------|--------|
| container | 容器 | systemd-nspawn, Docker, LXC 等 |
| vm | 虚拟机 | QEMU/KVM, libvirt 等 |
| host | 宿主 | 系统自身（特殊条目） |

### 2.4 管理命令（machinectl）

```bash
machinectl list                  # 列出运行中的机器
machinectl status <name>         # 查看机器状态
machinectl login <name>          # 登录容器
machinectl shell <name>          # 在容器中运行 shell
machinectl poweroff <name>       # 关闭容器
machinectl pull-tar <url>        # 下载 tar 镜像
machinectl pull-raw <url>        # 下载 raw 镜像
machinectl list-images           # 列出可用镜像
```

## 三、systemd-portabled — 便携服务

### 3.1 概述

`systemd-portabled` 管理"便携服务"——可以从磁盘镜像（raw/directory）中挂载服务文件到系统中，实现服务的便携部署。

D-Bus 名称：`org.freedesktop.portable1`
源码位于：`src/portable/`

### 3.2 工作原理

```
便携服务镜像（.raw / 目录）
  │
  │ portablectl attach
  ↓
systemd-portabled
  ├── 解剖镜像（dissect-image）
  │     解析 GPT 分区表
  │     找到根分区
  ├── 提取单元文件
  │     从镜像中读取 *.service 等文件
  ├── 安装到系统
  │     创建单元文件的 drop-in 或符号链接
  │     绑定挂载镜像内容到 RootImage=
  └── 可以正常使用 systemctl start/stop/enable
```

### 3.3 便携服务特点

- **独立性**：服务的所有文件打包在一个镜像中
- **隔离性**：通过 RootImage= 实现文件系统隔离
- **便携性**：镜像可以在不同机器间移动
- **安全性**：支持 Verity 完整性验证

### 3.4 管理命令

```bash
portablectl attach image.raw      # 附加便携服务
portablectl detach image.raw      # 分离便携服务
portablectl list                  # 列出已附加的镜像
portablectl inspect image.raw     # 检查镜像内容
portablectl reattach image.raw    # 重新附加（更新）
```
