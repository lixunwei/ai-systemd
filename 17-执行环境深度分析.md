# 执行环境深度分析 — execute.c：命名空间、沙箱与安全策略

> `exec_child()` 是 systemd 中最大的单体函数之一（~1070 行），负责在 fork() 后的
> 子进程中搭建完整的执行环境，包括命名空间隔离、挂载沙箱、权限降级、MAC/seccomp
> 安全策略，最终 execve() 目标程序。本文档基于 systemd v250 源码分析。

## 二、exec_spawn() — 进程创建

`execute.c:5084-5200`：

```
exec_spawn(unit, command, context, params, runtime, dcreds, &pid)
  │
  ├── 验证 socket FD（如果 stdin/stdout/stderr 连接到 socket）
  ├── 加载命名 IO FD
  ├── 加载 EnvironmentFile= 中的环境变量
  │
  ├── SELinux 数据库刷新
  │
  ├── 创建子 cgroup（如果 Delegate= 且 IS_CONTROL）
  │     → /sys/fs/cgroup/.../unit.service/.control
  │

  │                   → execve() 或 _exit(exit_status)
  │
  └── 父进程 → 返回 pid
```

## 三、exec_child() 完整执行流程

`execute.c:4011-5079`，约 1070 行。以下是按执行顺序的 **30 步** 完整流程：

```
exec_child()  [在 fork() 后的子进程中执行]
  │
  ├── [1] 信号重置
  │     → 恢复 SIGCRASH 等信号为 SIG_DFL
  │     → 忽略 SIGPIPE（如果 IgnoreSIGPIPE=yes）
  │     → 重置信号掩码
  │
  ├── [2] Idle 管道同步
  │     → Type=idle 服务等待管道，避免启动时输出混乱
  │
  ├── [3] 关闭不需要的 FD
  │     → 保留: socket FD、storage FD、exec_fd、BPF map fd
  │     → 关闭: 其他所有 FD（包括日志 FD）
  │     → 日志切换为 open_when_needed 模式
  │
  ├── [4] 创建新会话
  │     → setsid()（除非 SameProcessGroup=yes）
  │
  ├── [5] 终端设置
  │     → TTY 重置/挂断/释放
  │
  ├── [6] 确认执行
  │     → ConfirmSpawn= 交互式确认
  │
  ├── [7] 用户/组解析
  │     ├── DynamicUser=yes → dynamic_creds_realize()
  │     │     → 动态分配 UID/GID（61184-65519 范围）
  │     └── User=/Group= → get_fixed_user/get_fixed_group
  │     → 获取 supplementary groups
  │     → 通过 socketpair 发送 UID/GID 给 PID 1
  │
  ├── [8] 标准 IO 设置
  │     → setup_input():  stdin ← socket/null/tty/fd/file/data
  │     → setup_output(): stdout → journal/socket/tty/fd/file/inherit
  │     → setup_output(): stderr → 同上
  │
  ├── [9] cgroup 迁移
  │     → cg_attach_everywhere(cgroup_path, 0)
  │     → 从 init.scope 迁移到目标 cgroup
  │
  ├── [10] 命名空间路径
  │     → NetworkNamespacePath= → open_shareable_ns_path(CLONE_NEWNET)
  │     → IPCNamespacePath= → open_shareable_ns_path(CLONE_NEWIPC)
  │
  ├── [11] 进程属性设置
  │     → OOMScoreAdjust=
  │     → CoredumpFilter=
  │     → Nice=
  │     → CPUSchedulingPolicy=/Priority=
  │     → CPUAffinity=/NUMAPolicy=
  │     → IOSchedulingClass=/Priority=
  │     → TimerSlackNSec=
  │     → Personality=
  │
  ├── [12] utmp 记录
  │
  ├── [13] 终端所有权
  │     → chown(STDIN_FILENO, uid)
  │
  ├── [14] cgroup 委托
  │     → Delegate=yes → cg_set_access(cgroup_path, uid, gid)
  │
  ├── [15] 执行目录设置
  │     → StateDirectory=, RuntimeDirectory=, CacheDirectory=,
  │       LogsDirectory=, ConfigurationDirectory=
  │     → 创建目录、设置权限
  │
  ├── [16] 凭据设置
  │     → SetCredential=, LoadCredential=
  │     → 写入 /run/credentials/unit.service/
  │
  ├── [17] 环境变量构建
  │     → 基础环境（PATH, HOME, USER, SHELL...）
  │     → LISTEN_PID/LISTEN_FDS/LISTEN_FDNAMES（socket activation）
  │     → JOURNAL_STREAM, NOTIFY_SOCKET, WATCHDOG_*
  │     → PassEnvironment= + EnvironmentFile= + Environment=
  │
  ├── [18] umask 设置
  │     → UMask=（默认 0022）
  │
  ├── [19] 内核 keyring 设置
  │     → KeyringMode=inherit/private/shared
  │
  ├── ─── 判断沙箱需求 ─────────────────────────
  │     needs_sandboxing  = EXEC_APPLY_SANDBOXING && !FULLY_PRIVILEGED
  │     needs_setuid      = 同上 && !NO_SETUID && !ambient_hack
  │     needs_mount_ns    = exec_needs_mount_namespace()
  │     needs_ambient_hack= 内核不支持 ambient caps 的降级方案
  │
  ├── [20] MAC 检测（在新 mount ns 前）
  │     → SELinux: mac_selinux_use()
  │     → SMACK:   mac_smack_use()
  │     → AppArmor: mac_apparmor_use()
  │
  ├── [21] 资源限制（第一轮）
  │     → setrlimit() 所有 RLIMIT_* 配置
  │     → PAM 在此后调用，pam_limits 可以覆盖
  │
  ├── [22] PAM 会话设置
  │     → setup_pam(pam_name, username, uid, gid, tty)
  │     → 启动 PAM 代理子进程
  │
  ├── [23] 用户命名空间（提前）
  │     → PrivateUsers=yes 且无 CAP_SYS_ADMIN → 先设置 user ns
  │     → setup_private_users(): unshare(CLONE_NEWUSER)
  │     → 通过临时子进程写 uid_map/gid_map
  │
  ├── [24] 网络/IPC 命名空间
  │     → PrivateNetwork=yes → setup_shareable_ns(CLONE_NEWNET)
  │     → PrivateIPC=yes → setup_shareable_ns(CLONE_NEWIPC)
  │     → 命名空间通过 socketpair 在同 unit 的进程间共享
  │
  ├── [25] 挂载命名空间（核心沙箱）
  │     → apply_mount_namespace()
  │     → 详见第四节
  │
  ├── [26] UTS 命名空间
  │     → ProtectHostname=yes → unshare(CLONE_NEWUTS)
  │
  ├── [27] 组权限降级
  │     → enforce_groups(gid, supplementary_gids)
  │     → setresgid(gid, gid, gid)
  │     → setgroups(ngids, gids)
  │
  ├── [28] 用户命名空间（延后）
  │     → 有 CAP_SYS_ADMIN 时在此设置（晚于 mount ns 更优）
  │
  ├── [29] 查找可执行文件
  │     → find_executable_full(command->path, exec_search_path)
  │     → 在 mount namespace 内搜索
  │
  ├── [30] 安全策略（最后阶段）
  │     ├── SELinux: setexeccon(selinux_context)
  │     ├── AppArmor: aa_change_onexec(apparmor_profile)
  │     ├── Securebits: PR_SET_SECUREBITS
  │     ├── NoNewPrivileges=yes: prctl(PR_SET_NO_NEW_PRIVS)
  │     │
  │     ├── seccomp 过滤（按序应用）：
  │     │     [a] RestrictAddressFamilies=
  │     │     [b] MemoryDenyWriteExecute=yes
  │     │     [c] RestrictRealtime=yes
  │     │     [d] RestrictSUIDSGID=yes
  │     │     [e] RestrictNamespaces=
  │     │     [f] ProtectKernelTunables=yes（sysctl 写保护）
  │     │     [g] ProtectKernelModules=yes（模块加载禁止）
  │     │     [h] ProtectKernelLogs=yes（内核日志保护）
  │     │     [i] ProtectClock=yes（时钟修改禁止）
  │     │     [j] PrivateDevices=yes（设备节点 seccomp 层）
  │     │     [k] SystemCallArchitectures=
  │     │     [l] LockPersonality=yes
  │     │     [m] SystemCallLog=
  │     │     [n] SystemCallFilter=  ← 最后一个！
  │     │
  │     └── BPF: RestrictFileSystems=（LSM BPF）
  │
  ├── [31] 最终环境变量处理
  │     → UnsetEnvironment= 删除指定变量
  │     → 变量展开（$VAR 替换）
  │
  ├── [32] exec_fd 通知
  │     → write(exec_fd, 1) 通知 PID 1 即将 execve
  │
  └── [33] execve()
        → fexecve_or_execve(executable_fd, argv, env)
        → 如果失败 → write(exec_fd, 0) 通知 PID 1
        → _exit(EXIT_EXEC)
```

### 4.2 ProtectSystem= 保护级别

ProtectSystem=     │ 效果

### 4.4 ProtectKernelTunables=yes

## 五、命名空间隔离

### 5.1 七种命名空间

| 配置指令 | 命名空间 | clone 标志 | 效果 |
|----------|---------|-----------|------|
| PrivateUsers=yes | User NS | CLONE_NEWUSER | UID/GID 映射隔离 |
| PrivateNetwork=yes | Network NS | CLONE_NEWNET | 独立网络栈 |
| PrivateIPC=yes | IPC NS | CLONE_NEWIPC | 独立 SysV IPC/POSIX MQ |
| ProtectHostname=yes | UTS NS | CLONE_NEWUTS | 独立主机名 |
| PrivateMounts=yes | Mount NS | CLONE_NEWNS | 独立挂载点视图 |
| PrivatePIDs=yes *(未实现)* | PID NS | CLONE_NEWPID | — |
| ProtectControlGroups=yes | — | — | /sys/fs/cgroup 只读（挂载层面） |

### 5.2 用户命名空间实现

`setup_private_users()`（`execute.c:2110-2175`）：

```
setup_private_users(ouid, ogid, uid, gid)
  │
  ├── 构建 uid_map：
  │     $OUID → $OUID   (原始 UID 映射到自身)
  │     $UID  → $UID    (目标 UID 映射到自身，如果有 CAP_SETUID)
  │
  ├── 构建 gid_map（同理）
  │
  ├── fork() 临时子进程（保持在原 user ns 中）
  │     子进程等待父进程 unshare 完成
  │     子进程写入 /proc/$PPID/uid_map
  │     子进程写入 /proc/$PPID/gid_map
  │
  └── 父进程 unshare(CLONE_NEWUSER)
        通过 eventfd 通知子进程
        等待子进程完成映射写入
```

**为什么需要临时子进程？** 因为 unshare(CLONE_NEWUSER) 后失去了在原命名空间
写 uid_map 的权限（需要 CAP_SETUID in original user ns）。

### 5.3 命名空间共享

同一 Unit 的多个进程（主进程、ExecStartPre、ExecReload 等）通过 socketpair

## 六、seccomp 系统调用过滤

### 6.1 SystemCallFilter= 实现

`apply_syscall_filter()`（`execute.c:1485-1515`）：

```c
// 允许列表模式（SystemCallFilter=@system-service）：
default_action = SCMP_ACT_ERRNO(errno) 或 kill  // 未列出的 → 拒绝
action = SCMP_ACT_ALLOW                          // 列出的 → 允许

// 拒绝列表模式（SystemCallFilter=~@raw-io）：
default_action = SCMP_ACT_ALLOW                  // 未列出的 → 允许
action = SCMP_ACT_ERRNO(errno) 或 kill           // 列出的 → 拒绝
```

### 6.2 seccomp 策略应用顺序

在 `exec_child()` 中严格按以下顺序应用（`execute.c:4917-5011`）：

```
1. RestrictAddressFamilies=   → 限制 socket 地址族
2. MemoryDenyWriteExecute=yes → 禁止 W+X 内存映射
3. RestrictRealtime=yes       → 禁止实时调度 syscall
4. RestrictSUIDSGID=yes       → 禁止创建 SUID/SGID 文件
5. RestrictNamespaces=        → 限制 clone/unshare 的 NS 标志
6. ProtectKernelTunables=yes  → 禁止 sysctl 写入相关 syscall
7. ProtectKernelModules=yes   → 禁止 init_module/delete_module
8. ProtectKernelLogs=yes      → 禁止 syslog() syscall
9. ProtectClock=yes           → 禁止 settimeofday/clock_settime
10. PrivateDevices=yes        → 禁止 ioctl 中的设备相关操作
11. SystemCallArchitectures=  → 限制允许的 CPU 架构
12. LockPersonality=yes       → 禁止 personality() 切换
13. SystemCallLog=            → 记录指定系统调用
14. SystemCallFilter=         → 最终过滤（最后，影响最小化）
```

**设计原则**：SystemCallFilter= 放在最后，确保前面所有 systemd 自身的
syscall（设置 seccomp 规则本身就需要 syscall）都不受影响。

### 6.3 BPF LSM: RestrictFileSystems=

`execute.c:5005-5011`：

```c

## 七、MAC 强制访问控制

### 7.1 SELinux

```
两个来源的安全上下文：
1. SELinuxContext= → 配置文件指定
2. 网络对等方上下文 → socket activation 时从对端 socket 获取

应用时机：execve() 前通过 setexeccon() 设置
  → 不立即生效
  → execve() 时内核自动切换到新上下文

### 7.2 AppArmor

```
AppArmorProfile= → aa_change_onexec(profile)
  → 类似 SELinux，在 execve() 时切换到新 profile
```

### 7.3 SMACK

```
SmackProcessLabel= → 通过 /proc/self/attr/current 设置
  → 必须在 execve() 前完成
  → 需要 CAP_MAC_ADMIN（在 capability 降级前设置）
```

## 八、Capabilities 管理

### 8.1 三层 capabilities

```
exec_child() 中的 capability 处理：

1. CapabilityBoundingSet=
   → capability_bounding_set_drop(bset)
   → 限制进程可获取的最大 capability 集合

2. AmbientCapabilities=
   → capability_ambient_set_apply()
   → 在 setresuid() 前设置 inheritable 集
   → 在 setresuid() 后重新设置 ambient 集
   → 无需 root 即可保留 capabilities

3. SecureBits=
   → prctl(PR_SET_SECUREBITS, bits)
   → keep-caps, no-setuid-fixup, noroot 等
```

### 8.2 Ambient Capabilities Hack

```
旧内核不支持 ambient caps 时的降级方案：
  → 在 bounding set 中额外保留 CAP_SETPCAP/SETUID/SETGID
  → 在 seccomp 白名单中添加 SYSCALL_FILTER_SET_SETUID
  → 让服务自己完成权限降级
```

## 九、exec_fd 通知机制

`execute.c:5051-5075`：

```c
// exec_fd 是一个设置了 O_CLOEXEC 的管道
// PID 1 通过 epoll 监控 exec_fd

// execve() 成功时：
//   → O_CLOEXEC 自动关闭 exec_fd
//   → PID 1 收到 POLLHUP → 知道 execve 成功

// execve() 失败时：
//   → write(exec_fd, 0)  → 通知 PID 1 execve 失败
//   → PID 1 不把 POLLHUP 当作成功

// execve() 前：
//   → write(exec_fd, 1)  → 通知 PID 1 即将 execve
```

## 十、DynamicUser= 动态用户

```
DynamicUser=yes：
  │
  ├── UID/GID 范围：61184-65519（DYNAMIC_UID_MIN-DYNAMIC_UID_MAX）
  │
  ├── 分配机制：
  │     → dynamic_creds_realize() 使用随机起点循环搜索可用 UID
  │     → 通过 /run/systemd/dynamic-uid/ 下的锁文件独占 UID
  │
  ├── 自动启用：
  │     → PrivateTmp=yes（隔离 /tmp）
  │     → RemoveIPC=yes（停止时清理 IPC）
  │     → ProtectHome=yes（隐藏 /home）
  │     → ProtectSystem=strict（只读文件系统）
  │
  └── 目录所有权：
        → StateDirectory= 等目录 chown 给动态用户
        → 停止时 UID 释放，但目录保留
```
